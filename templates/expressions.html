The primary syntactic construct in XPath is the expression.

The most important kind of expression is a location path.


An expression is evaluated to yield an object

Expression evaluation occurs with respect to a context.

Functions are expressions that accept zero or more arguments and return a single result. See the function reference.

The value of a variable is an object.

Arguments, results and variables must one of the 4 basic types.

Namespaces are mapped from prefixes to URIs

XML special chars must be "escaped" e.g. &lt;

Literal strings must be quoted with " or '

Use &quot; or &apos; to avoid confusion with the end of an attribute. Or use alternating single and double quotes.

Whitespace is allowed between expression tokens.

Subexpressions

Variable bindings, function library and namespace are the same as containing function.

The context node, position and size may be different, however. Several kinds of expressions change the context node. Only predicates change the context position and size.
