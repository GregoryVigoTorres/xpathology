The primary syntactic construct in XPath is the expression.

The most important kind of expression is a location path.


An expression is evaluated to yield an object

Expression evaluation occurs with respect to a context.

WTF is an expression and how do I use one?

The value of a variable is an object. Like with expression value types.

Functions accept 0 or more arguments and return a single result. See core function library.

Arguments and results must one of the 4 basic types.

Namespace is mapping from prefixes to URIs

XML special chars must be "escaped" e.g. &lt;

Literal strings must be quoted with " or '

Use &quot; or &apos; to avoid confusing end of attribute. Or use alternating single and double quotes.

Whitespace is allowed between expression token

Subexpressions

Variable bindings, function library and namespace are the same as containing function.

The context node, position and size may be different. Several kinds of expressions change the context node. Only predicates change the context position and size.
